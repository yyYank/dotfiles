# Review Pitfall Checker

レビューで抜け漏れがちな観点をチェックします。

## チェック観点

### バージョンの妥当性
- 安定最新バージョンを使用しているか
- LLMのスコアリングで古いバージョンを推奨・設定していないか
- 例: Postgres 18が安定版なのに15を指定している

### 機密情報の取り扱い
- 平文で管理すべきでないものを管理していないか
- パスワード、APIキー、トークン等を変数として平文で扱っていないか
- 環境変数やシークレット管理サービスを使用しているか

### TODO/FIXME の管理
- TODO実装が意図せずコードに混在していないか
- docs や GitHub Issue に転記済みか
- 一時的な実装が本番コードに残っていないか

## 出力形式

チェック結果は以下の形式で出力してください：

```
## 概要
[チェック対象の概要と全体評価]

## 検出した落とし穴

### [リスク: 高/中/低] [カテゴリ]
- ファイル: [ファイルパス:行番号]
- 問題: [見落としがちなポイント]
- 想定シナリオ: [問題が顕在化するケース]
- 対策案: [改善方法]

## 確認済み項目
[問題なしと判断した観点があれば記載]
```
