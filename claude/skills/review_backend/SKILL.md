# Backend Code Review

バックエンドコードのレビューを行います。

## レビュー観点

### アーキテクチャ・設計
- レイヤー構造の適切さ（Controller, Service, Repository等）
- 責務の分離ができているか
- 依存関係の方向は正しいか

### セキュリティ
- SQLインジェクション対策
- 認証・認可の実装
- 入力値のバリデーション
- 機密情報の取り扱い

### パフォーマンス
- N+1問題の有無
- 適切なインデックス利用
- キャッシュ戦略
- 非同期処理の活用

### エラーハンドリング
- 例外処理の適切さ
- エラーメッセージの適切さ
- リトライ戦略

### コード品質
- 命名規則の一貫性
- DRY原則の遵守
- 適切なログ出力
- テスタビリティ

## 出力形式

レビュー結果は以下の形式で出力してください：

```
## 概要
[全体的な評価]

## 指摘事項

### [重要度: 高/中/低] [カテゴリ]
- ファイル: [ファイルパス:行番号]
- 問題: [問題の説明]
- 提案: [改善案]

## 良い点
[コードの良い点があれば記載]
```
